**TRABALHO EM GRUPO: SEMINÁRIO DE SISTEMAS OPERACIONAIS**

**Como vai funcionar?**

Nesta atividade, cada grupo vai escolher um Sistema Operacional e montar uma aula para apresentar para o resto da turma. O objetivo não é ler slides para o professor, mas sim explicar para os seus colegas como esse sistema funciona na prática, para quem ele serve e o que ele tem por baixo do capô.

**A Escolha do Sistema Operacional**

O tema é livre, desde que seja um sistema voltado para **Desktop (computadores de mesa ou notebooks)**. Vocês podem escolher desde os mais tradicionais do mercado, como Windows e macOS, até explorar o universo das distribuições Linux (como Fedora, Ubuntu, Linux Mint, Pop!_OS, Arch, etc.).

A única regra é: não teremos sistemas repetidos. Os grupos precisam conversar e definir quem fica com qual.

**O que vocês precisam explicar**

Para a apresentação não virar apenas um artigo da Wikipédia lido em voz alta, a aula de vocês precisa cobrir obrigatoriamente os seguintes pontos:

- **Contexto e Propósito:** Quando surgiu? Qual era a ideia principal dos criadores na época?
    
- **Por baixo dos panos:** Qual é o tipo de Kernel utilizado? Qual é o sistema de arquivos padrão daquele sistema (ex: NTFS, ext4, APFS)? Como ele gerencia os recursos da máquina?
    
- **A Cara do Sistema:** Como é a interface e a usabilidade diária para o usuário? É fácil de aprender?
    
- **Prós e Contras:** Quais são as maiores vantagens e as maiores dores de cabeça desse sistema? (Sejam críticos, analisem os defeitos também).
    
- **Público-Alvo:** Para quem esse sistema é mais indicado hoje em dia? (Gamers, desenvolvedores, usuários casuais, designers?)
    

**A Prática: Mostrando o Sistema Rodando**

Falar de sistema operacional sem mostrar a tela não funciona. O grupo precisa reservar uns 5 minutinhos da apresentação para mostrar o sistema operando de verdade.

Vocês podem trazer um notebook rodando o sistema nativamente, fazer um dual-boot, ou subir uma Máquina Virtual (VirtualBox, VMware, GNOME Boxes, etc.). A ideia é abrir o sistema ao vivo, mostrar o gerenciador de tarefas/monitor do sistema, o terminal, navegar pelas configurações e explicar visualmente o que vocês falaram na teoria.

**Como a nota será calculada**

A avaliação (de 0 a 10) vai considerar quatro critérios básicos:

1. **Domínio Técnico (3,0 pontos):** O grupo entendeu os conceitos de kernel e sistema de arquivos, ou apenas decorou o texto do slide?
    
2. **Didática (2,5 pontos):** A aula foi clara, trouxe exemplos bons e conseguiu prender a atenção do resto da turma?
    
3. **Demonstração Prática (2,5 pontos):** O sistema foi mostrado rodando ao vivo com fluidez? Explicaram o que estava acontecendo na tela?
    
4. **Sabatina (2,0 pontos):** O grupo conseguiu desenrolar as respostas para as perguntas feitas pelo professor e pelos colegas no final da apresentação?
    

---

_Duração: 10 minutos de apresentação do grupo + 5 minutos de perguntas e discussão aberta._
